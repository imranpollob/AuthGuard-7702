### Exact AuthGuard estimator

- XGBoost classifier; **300** trees; maximum depth **6**; learning rate **0.1**; row subsampling **0.9**; column subsampling **0.8**; `tree_method="hist"`; `eval_metric="logloss"`; `n_jobs=4`; random seed **7702**.
- Positive-class `predict_proba` output is the score.
- AuthGuard-M0 has no explicit class-weight parameter and uses uniform sample weights. AuthGuard-aug uses source-balanced sample weights; this is not class reweighting.
- Sources: `pipeline/03_detection.py` (`gb_clf`, fit, score, and training-threshold code), `pipeline/adv_run.py` (`XGB_HP` and weighted fit), and `paper_build/data_hygiene/task_aligned_rerun.py` (task-aligned reuse).

## Exact protocols and mutation definitions

- **G-DET:** primary malicious-versus-`benign_cleared`; five preserved family outer folds; four folds fit and one tests; maximum-F1 threshold from fitting predictions; seeded random five-fold KFold only as a diagnostic; AUPRC primary.
- **G-MUT:** clean M0 models; held-out positives transformed after family splitting; cumulative M0–M3; retained recall.
- **G-VOL:** metadata/address/selector compound recipe plus variable appended-dead-code volume. It remains distinct from G-ADV F200.
- **G-ADV:** test fold `f`, clean-M0 validation fold `(f+1) mod 5`, remaining three fitting folds; validation maximum-F1 threshold; training conditions M0/M1/M2/F25/F50/F100; held-out M3 and pure-M0 F200; separate deterministic train/test RNG domains; both labels augmented symmetrically; source-balanced weights.

Sources: `paper_build/data_hygiene/task_alignment_protocol.md`, `task_aligned_rerun.py`, `pipeline/03_detection.py`, `pipeline/04_mutations.py`, and `pipeline/adv_run.py`.

Implemented transformations were read from `pipeline/04_mutations.py`, not copied from an old report:

- **M0:** normalized original bytecode.
- **M1:** rewrite bytes in a detected `0xa2` CBOR-style trailer except its marker and final length bytes; if absent, append a synthetic CBOR-like trailer.
- **M2:** cumulative M1; randomize pre-metadata PUSH20 immediates; append STOP and a deterministic benign-sourced byte chunk of about **20%** of the original executable-byte length.
- **M3:** cumulative M2; run the selector-rewrite scan on the transformed byte array and rewrite PUSH4 immediates that match the sensitive-selector set. Because M2 has appended bytes after the metadata trailer, the helper can treat the whole transformed array as its scan region; the paper does not incorrectly call this a pre-metadata-only rewrite.
- **F25/F50/F100/F200:** append STOP and benign-sourced bytes sized to **25%/50%/100%/200%** of the original executable-byte length. G-ADV applies these directly to M0; G-VOL separately composes metadata/address/selector changes with the selected volume.
- Checker scope: compare opcode-token sequences only within the original pre-metadata region. **727/727** retained positives passed at each of M1, M2, and M3. This is explicitly not an execution- or behavioral-equivalence proof.

Source-balanced augmentation uses six variants per source. Each variant receives `1/n_i`; all variants of source `i` sum to one source-contract unit. This is symmetric across benign and malicious sources. The prose limits its shortcut claim to variant-count and padding-label shortcuts and does not claim that every shortcut is absent.

## Unresolved implementation and TeX concerns

- The feature code builds the sensitive-selector set from hard-coded signatures plus an available artifact file; the frozen feature metadata records columns but not a separate hash of that selector set. The section therefore describes its category and code provenance without claiming an independently versioned selector vocabulary.
- Appended bytes are treated by the harness as dead code and preceded by STOP, but the checker does not prove dynamic unreachability or behavior preservation. The prose confines the guarantee to its pre-metadata opcode-token check.
- No TeX engine is installed, so page count and syntax could not be validated by compilation. `git diff --check` passed.
- `system_architecture.tex` requires `\usepackage{tikz}` in the manuscript preamble; it deliberately uses no optional TikZ libraries. The current legacy `paper/authguard7702.tex` preamble does not load TikZ and was outside the authorized edit scope.
- Section inputs use repository-root-relative paths (`paper_build/figures/...` and `paper_build/tables/...`). A final driver compiled from another working directory must adjust `\input@path` or these two paths.
- The table requires `booktabs` and `array`; both are already present in the legacy IEEE driver.
